title: "Building FRR"
description: "Build FRR from source: dependencies, configure options, and compilation"
---

FRR (FRRouting) is built from source using the standard autotools workflow. This guide covers the full build process including dependencies, configure flags, compilation, and post-install setup.

## Prerequisites

Before building FRR, you need the following tools and libraries installed on your system:

| Category | Packages |
|---|---|
| Build tools | `autoconf`, `automake`, `libtool`, `make`, `gcc`, `bison`, `flex` |
| Development libs | `libreadline-dev`, `libc-ares-dev` |
| Python | `python3`, `python3-dev`, `python3-sphinx` |
| Networking | `libyang` (see below), `libsnmp-dev` |
| Optional | `libprotobuf-c-dev`, `libzmq3-dev` |

<Note>
`libyang` must be built from source on most platforms. See the platform-specific pages for exact build instructions.
</Note>

<Steps>

<Step title="Install dependencies">

Install the required packages for your platform. The exact package names vary by distribution — see [Platform-Specific Build Instructions](/developer/building-platforms) for Debian/Ubuntu and CentOS details.

On Ubuntu 18.04 or later:

```bash
sudo apt-get install \
  git autoconf automake libtool make libreadline-dev texinfo \
  pkg-config libpam0g-dev libjson-c-dev bison flex python-pytest \
  libc-ares-dev python3-dev libsystemd-dev python3-sphinx \
  install-info build-essential libsnmp-dev perl
```

</Step>

<Step title="Install libyang">

FRR requires `libyang` for its YANG/northbound API. Build it from source:

```bash
git clone https://github.com/CESNET/libyang.git
cd libyang
git checkout v1.0.184
mkdir build && cd build
cmake -DENABLE_LYD_PRIV=ON -DCMAKE_INSTALL_PREFIX=/usr \
  -D CMAKE_BUILD_TYPE:String="Release" ..
make
sudo make install
```

</Step>

<Step title="Clone the FRR repository">

```bash
git clone https://github.com/frrouting/frr.git frr
cd frr
```

</Step>

<Step title="Run bootstrap.sh">

`bootstrap.sh` generates the `configure` script from the autotools template files. This step is required when building from a git checkout (as opposed to a release tarball, which ships a pre-generated `configure`).

```bash
./bootstrap.sh
```

</Step>

<Step title="Create the frr user and group">

FRR daemons run as a dedicated system user. Create the user and groups before installing:

```bash
sudo groupadd -r -g 92 frr
sudo groupadd -r -g 85 frrvty
sudo adduser --system --ingroup frr --home /var/run/frr/ \
  --gecos "FRR suite" --shell /sbin/nologin frr
sudo usermod -a -G frrvty frr
```

</Step>

<Step title="Configure the build">

Run `./configure` with flags appropriate to your system. The options below represent a typical production-oriented build:

```bash
./configure \
    --prefix=/usr \
    --includedir=${prefix}/include \
    --enable-exampledir=${prefix}/share/doc/frr/examples \
    --bindir=${prefix}/bin \
    --sbindir=${prefix}/lib/frr \
    --libdir=${prefix}/lib/frr \
    --libexecdir=${prefix}/lib/frr \
    --localstatedir=/var/run/frr \
    --sysconfdir=/etc/frr \
    --with-moduledir=${prefix}/lib/frr/modules \
    --with-libyang-pluginsdir=${prefix}/lib/frr/libyang_plugins \
    --enable-configfile-mask=0640 \
    --enable-logfile-mask=0640 \
    --enable-snmp=agentx \
    --enable-multipath=64 \
    --enable-user=frr \
    --enable-group=frr \
    --enable-vty-group=frrvty \
    --with-pkg-git-version \
    --with-pkg-extra-version=-MyOwnFRRVersion
```

<Note>
If your platform uses `systemd`, add `--enable-systemd=yes` to your configure options.
</Note>

</Step>

<Step title="Compile and install">

```bash
make
make check
sudo make install
```

`make check` runs the test suite. It is optional but recommended before deployment.

</Step>

<Step title="Set up /etc/frr">

Install configuration files and set correct ownership:

```bash
sudo install -m 775 -o frr -g frr -d /var/log/frr
sudo install -m 775 -o frr -g frrvty -d /etc/frr
sudo install -m 640 -o frr -g frrvty tools/etc/frr/vtysh.conf /etc/frr/vtysh.conf
sudo install -m 640 -o frr -g frr tools/etc/frr/frr.conf /etc/frr/frr.conf
sudo install -m 640 -o frr -g frr tools/etc/frr/daemons.conf /etc/frr/daemons.conf
sudo install -m 640 -o frr -g frr tools/etc/frr/daemons /etc/frr/daemons
```

</Step>

<Step title="Install systemd service and enable daemons">

```bash
sudo install -m 644 tools/frr.service /etc/systemd/system/frr.service
sudo systemctl enable frr
```

Open `/etc/frr/daemons` in an editor and set the daemons you want to run to `yes`. For example:

```
zebra=yes
bgpd=yes
ospfd=no
```

Then start FRR:

```bash
systemctl start frr
```

</Step>

</Steps>

## Key configure flags

The full list of configure options is available via `./configure -h`. The most important flags are described below.

### Path flags

| Flag | Default | Description |
|---|---|---|
| `--prefix` | `/usr/local` | Top-level installation prefix |
| `--sysconfdir` | `${prefix}/etc` | Location for config files (`/etc/frr`) |
| `--localstatedir` | `${prefix}/var` | Runtime state directory (`/var/run/frr`) |
| `--sbindir` | `${prefix}/sbin` | Location for daemon binaries |

### User and group flags

| Flag | Description |
|---|---|
| `--enable-user=frr` | Run daemons as this system user |
| `--enable-group=frr` | Primary group for daemon processes |
| `--enable-vty-group=frrvty` | Group for VTY socket access |

### Feature flags

| Flag | Description |
|---|---|
| `--enable-multipath=256` | Sets the maximum number of ECMP paths (default 64) |
| `--enable-snmp=agentx` | Enable SNMP support via AgentX protocol |
| `--with-pkg-git-version` | Embed the current git hash in the binary version string |
| `--enable-bgp-vnc` | Enable BGP VNC (Virtual Network Control) support |
| `--enable-fpm` | Enable the Forwarding Plane Manager interface |
| `--enable-systemd=yes` | Enable systemd integration for watchfrr |
| `--enable-dev-build` | Enable developer-mode assertions and additional diagnostics |

### Protocol-specific flags

Individual protocol daemons can be disabled at configure time:

```bash
--disable-ldpd       # exclude LDP daemon
--disable-nhrpd      # exclude NHRP daemon
--disable-bgpd       # exclude BGP daemon
```

## Enabling IP forwarding

FRR requires kernel IP forwarding to be enabled. Edit `/etc/sysctl.conf` and uncomment (or add) these lines:

```
net.ipv4.ip_forward=1
net.ipv6.conf.all.forwarding=1
```

Apply immediately without rebooting:

```bash
sudo sysctl -p
```

## Verifying the build

After installation, check that the daemons are available and display their version:

```bash
/usr/lib/frr/zebra --version
/usr/lib/frr/bgpd --version
```

Use `vtysh` to connect to the running FRR process:

```bash
sudo vtysh
```

From the vtysh prompt, `show version` confirms the build flags and git commit embedded at compile time.
