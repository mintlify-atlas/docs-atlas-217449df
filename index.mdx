FRRouting (FRR) is a free, open-source routing software suite that implements and manages IPv4 and IPv6 routing protocols. The sonic-frr fork is tailored for [SONiC](https://sonic-net.github.io/SONiC/) — the open-source network operating system — and runs on nearly all Linux distributions and BSD variants.

FRR uses a multi-daemon architecture: each routing protocol runs as an independent daemon, all coordinated by the **zebra** kernel routing manager. You interact with any daemon through the unified **vtysh** CLI shell.

## Getting started

<Steps>
  <Step title="Install FRR">
    Install packages for your Linux distribution, or build from source. See [Installation](/installation).
  </Step>
  <Step title="Enable daemons">
    Edit `/etc/frr/daemons` to activate the protocols you need (bgpd, ospfd, etc.). See [Basic Setup](/basic-setup).
  </Step>
  <Step title="Configure via vtysh">
    Launch `vtysh` to enter the unified CLI and configure your routing protocols interactively. See [vtysh CLI](/operations/vtysh).
  </Step>
  <Step title="Verify routing">
    Use `show ip route`, `show bgp summary`, or protocol-specific `show` commands to verify routing table state.
  </Step>
</Steps>

<Note>
  sonic-frr is a fork of the upstream FRRouting project maintained by the SONiC community. It tracks upstream FRR closely while adding SONiC-specific integrations.
</Note>
